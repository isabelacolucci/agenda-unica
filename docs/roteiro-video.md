# Roteiro para Gravação do Vídeo – Agenda Única

## 1. Introdução
- Apresentação pessoal: Nome, curso e disciplina.
- Apresentação do projeto: "Agenda Única" – solução desenvolvida para a disciplina Projeto de Extensão em Software Fullstack.

## 2. Problematização
- Contextualização do problema: dificuldade de profissionais autônomos e pequenos negócios em gerenciar agendamentos de serviços de forma eficiente.
- Impactos do problema: perda de clientes, conflitos de horários, baixa produtividade.
- Objetivo do software: oferecer uma plataforma simples, centralizada e acessível para gestão de agendas, serviços e atendimentos.

## 3. Solução Proposta
- Descrição do MVP: funcionalidades essenciais para validação da ideia (cadastro de usuários, serviços, agendamento, painel do prestador, página pública, etc).
- Diferenciais: interface intuitiva, responsividade, acesso facilitado para clientes e prestadores.

## 4. Metodologia e Estratégias de Desenvolvimento
- Utilização de técnicas de "vibe code" para acelerar a entrega do MVP.
- Apoio de agentes de IA (como o GitHub Copilot com agentes GPT e Claude Sonnet) para geração de código, automação de tarefas repetitivas e sugestões de boas práticas.
- Engenharia de prompt: elaboração de comandos e instruções detalhadas para orientar a IA na geração de código e documentação.
- Criação de planos e roteiros de desenvolvimento para garantir alinhamento e eficiência (exemplo: arquivos em `docs/plans/`).
- Documentação de apoio: decisões técnicas, requisitos, checklist de componentes, etc, para facilitar a colaboração e manutenção.

## 5. Demonstração do Sistema
- Apresentação da interface principal (homepage, login, cadastro).
- Demonstração do fluxo de agendamento de um serviço.
- Visualização do painel do prestador (gestão de serviços, horários, configurações).
- Exemplo de página pública do prestador.
- Destaque para recursos de usabilidade e responsividade.

## 6. Considerações Finais
- Benefícios do sistema para o público-alvo.
- Potencial de evolução do projeto.
- Agradecimento e convite para acessar o repositório e testar o sistema.
